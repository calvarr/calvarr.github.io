Each edition is optimized for a different usage style.

Desktop — recommended

DLPulse Next

Cross-platform desktop shell for educational use: resolves public page URLs via yt-dlp, local library, playback, Chromecast, format presets. Uses ffmpeg where needed. Native UI via pywebview (WebKitGTK on Linux, WebView2 on Windows, WKWebView on macOS).

Install DLPulse Next

Pre-built installers for Windows and macOS. On Linux, build a Flatpak locally (recommended) or install from source with your distro packages (GTK + WebKit + GStreamer).

Download releases Continuous build: DLPulseNext-Setup.exe (Windows), DLPulseNext-x86_64.dmg / DLPulseNext-arm64.dmg (macOS)

Linux — Flatpak (recommended)

Sandboxed desktop bundle using the GNOME runtime (GTK3 + WebKit2GTK + GStreamer). Build locally from a git clone — not on Flathub yet. Bundled: yt-dlp, aria2c; ffmpeg via imageio-ffmpeg on first use.

1. Flatpak tools (one-time)

Arch / Manjaro:

sudo pacman -S --needed flatpak flatpak-builder

Debian / Ubuntu:

sudo apt install -y flatpak flatpak-builder

Fedora:

sudo dnf install -y flatpak flatpak-builder

2. GNOME runtime (one-time)

flatpak remote-add --if-not-exists flathub https://flathub.org/repo/flathub.flatpakrepo
flatpak install flathub org.gnome.Platform//49 org.gnome.Sdk//49

3. Build and run

git clone https://github.com/calvarr/DLPulse-next.git
cd DLPulse-next
bash packaging/flatpak/build.sh
flatpak run com.github.calvarr.DLPulseNext

Config and library paths use ~/.config/dlpulse-next (same as from-source). Details: packaging/flatpak/README.md

Linux — Arch / Manjaro / EndeavourOS (from source)

Requires Python 3.11+. Alternative to Flatpak — uses system WebKit for a native window.

1. System packages

sudo pacman -S --needed \
  python python-pip python-gobject \
  gtk3 webkit2gtk-4.1 gobject-introspection-runtime \
  gstreamer gst-plugins-base gst-plugins-good \
  mpv ffmpeg aria2

2. Application

git clone https://github.com/calvarr/DLPulse-next.git
cd DLPulse-next
python3 -m venv .venv
source .venv/bin/activate
pip install -U pip wheel
pip install -e ".[webview-gtk]"
dlpulse-next

Windows

  1. Download DLPulseNext-Setup.exe from GitHub Releases.
  2. Run the installer (installs under Program Files\DLPulse Next).
  3. Launch DLPulse Next from the Start menu.

Bundled: yt-dlp, ffmpeg, aria2c, WebView2 UI.

macOS — DLPulse Next

Ad-hoc signed (no Apple Developer ID). If macOS says the DMG or app is corrupt or damaged, that is usually Gatekeeper — not a broken file.

  1. Download DLPulseNext-x86_64.dmg (Intel) or DLPulseNext-arm64.dmg (Apple Silicon). Check architecture: uname -m → x86_64 = Intel, arm64 = Apple Silicon.
  2. If the DMG won’t mount (“corrupt” / “damaged”), clear quarantine on the downloaded file, then open it: 
    xattr -cr ~/Downloads/DLPulseNext-x86_64.dmg
    (Adjust the path if you saved the DMG elsewhere.)
  3. Open the DMG and drag DLPulseNext.app to Applications.
  4. After copying, remove quarantine on the app: 
    sudo xattr -r -d com.apple.quarantine "/Applications/DLPulseNext.app" 2>/dev/null
  5. Or: right‑click the app → Open, or System Settings → Privacy & Security → Open Anyway.

Bundled: yt-dlp, ffmpeg, aria2c, WKWebView UI.

Full details: README on GitHub (Flatpak + from source)

Install DLPulse Flet

Pre-built installers for Windows, macOS, and Linux (AppImage). On Linux you can also run from source with Python 3.11+ and your distro packages.

Download releases Continuous build: DLPulse-Setup.exe (Windows), DLPulse.dmg (macOS), DLPulse-x86_64.AppImage (Linux)

Windows — DLPulse Flet

  1. Download DLPulse-Setup.exe from GitHub Releases.
  2. Run the installer and launch DLPulse from the Start menu.

Bundled: yt-dlp, ffmpeg, Flet desktop UI.

macOS — DLPulse Flet

Ad-hoc signed (no Apple Developer ID). After copying to Applications, remove quarantine if macOS blocks launch.

  1. Download DLPulse.dmg from GitHub Releases.
  2. Open the DMG and drag DLPulse.app to Applications.
  3. Remove quarantine (recommended if blocked or “damaged”): 
    sudo xattr -r -d com.apple.quarantine "/Applications/DLPulse.app" 2>/dev/null
  4. Or: right‑click the app → Open, or System Settings → Privacy & Security → Open Anyway.

Bundled: yt-dlp, ffmpeg, Flet desktop UI.

Linux — AppImage (pre-built)

No install step — download, make executable, run. Alternative: from source.

  1. Download DLPulse-x86_64.AppImage from GitHub Releases.
  2. Make executable and run: 
    chmod +x DLPulse-x86_64.AppImage
./DLPulse-x86_64.AppImage

Bundled: yt-dlp, ffmpeg, Flet desktop UI.

Linux — from source

Requires Python 3.11+, mpv (in-app video via libmpv), and ffmpeg on PATH (yt-dlp merges). GTK3 is used by the Flet desktop shell.

1. System packages

Arch / Manjaro:

sudo pacman -S --needed python python-pip gtk3 mpv ffmpeg

Debian / Ubuntu:

sudo apt update
sudo apt install -y python3 python3-pip python3-venv libgtk-3-0 mpv ffmpeg

Fedora:

sudo dnf install -y python3 python3-pip gtk3 mpv ffmpeg

2. Clone and run

git clone https://github.com/calvarr/DLPulse.git
cd DLPulse
python3 -m venv .venv
source .venv/bin/activate
pip install -U pip wheel
pip install -e .
./flet_app/run.sh

pip install -e . installs Flet, yt-dlp, Chromecast deps, and imageio-ffmpeg. To build your own AppImage locally (on Linux): ./build_linux.sh --appimage.

Full details: README on GitHub
